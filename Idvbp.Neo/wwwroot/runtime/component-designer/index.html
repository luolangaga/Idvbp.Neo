积木怎么用

  • 当事件 是入口；把“设置元素文字”“移动元素”等操作接到它下面。
  • 房间数据 填上方路径，例如 teamA.name，可接到“设置元素文字”。
  • 事件数据 读取当前事件内容，例如 payload.map.name。
  • 循环 N 次 会重复执行内部积木；遍历列表 可遍历数组，并用“循环项”读取当前项。
  • 变量 是组件运行时的临时存储。用“设置变量 user.name 为 ...”，后面用“读取变量 user.name”取值。
  • 请求后端 API 是圆形取值积木，可以直接插到“设置变量”“设置元素文字”“遍历列表”等输入口。普通请求只填方法和路径。
  • 带 JSON 请求体的 API 只给 POST、PATCH、PUT 这类接口用。GET 房间列表不需要请求体。
  • API 返回 JSON 会自动解析成对象或数组。推荐先“设置变量 rooms 为 请求后端 API GET /api/rooms”，后续用“读取变量 rooms”复用。
  • 取对象路径 用来从 API 返回值、变量、房间数据里继续取字段，例如路径 0.roomName 或 data.items.0.name。
  • 组件配置 是持久化数据，刷新后还在；变量是临时数据，适合 API 结果、中间计算值、循环结果。

可调用的后端 API

GET /api/health检查本地后端是否正常。 GET /api/rooms读取全部 BP 房间。 GET /api/rooms/{roomId}读取指定房间快照。 POST /api/rooms创建房间，请求体可传 {"roomName":"房间名"}。 POST /api/rooms/{roomId}/matches创建/重置比赛信息。 PATCH /api/rooms/{roomId}/map更新地图选择和地图禁用。 POST /api/rooms/{roomId}/bans添加本局角色禁用。 POST /api/rooms/{roomId}/global-bans添加全局角色禁用。 POST /api/rooms/{roomId}/roles更新求生者/监管者选角。 PATCH /api/rooms/{roomId}/phase更新 BP 阶段。 PATCH /api/rooms/{roomId}/teams更新队伍名称、成员、阵营。 GET /api/signalr/events查看所有 SignalR 房间事件名。 GET /api/resources/characters读取角色资源列表。 GET /api/resources/characters/{characterId}读取单个角色资源。 GET /api/resources/maps读取地图资源列表。 GET /api/resources/maps/{mapId}读取单个地图资源。 GET /api/frontends读取已安装前台包。 GET /api/frontends/{id}读取指定前台包信息。 GET /api/frontends/{id}/pages/{pageId}/config读取页面配置。 GET /api/frontends/{id}/pages/{pageId}/components/config读取页面组件配置集合。 GET /api/runtime-logs读取运行时日志。 DELETE /api/runtime-logs清空运行时日志。 GET /api/local-bp-state读取本地 BP 聚合状态，适合 3D/覆盖层展示。 GET /api/official-model-map读取官方角色模型映射。

常见组合

  1. 当事件 room.snapshot -> 设置元素 title 文字为 房间数据 roomName
  2. 当事件 designer.map.selected -> 设置图片元素资源为 事件数据 payload.map.imageUrl
  3. 设置变量 rooms 为 请求后端 API GET /api/rooms -> 遍历列表“读取变量 rooms” -> 用“循环项 roomName”读取每个房间名。
  4. 设置元素文字为 取对象路径“请求后端 API GET /api/health”的 status。
  5. 设置变量 firstRoom 为 取对象路径“读取变量 rooms”的 0.roomName。
  6. 设置变量 title 为 房间数据 teamA.name -> 设置元素文字为 读取变量 title。
